hugo+GitHub入门

  2019-11-27

我的github

hugo简介

使用Go语言编写的静态站点生成器（注：官网文档 hugo中文网文档）

（本人是小白一个，请大家来指点提携）现在的博客主流框架有很多，比如hexo,docker也可用来搭建（动态）博客，简单来说一下hugo 因为本人是贫穷小白所以本文以介绍windows（为主）和linux

优缺点

  • 优点

  1. 体积小

  2. 安装步骤简单，适合小白

  3. 操作简单

  4. 运行快

  • 缺点
  1. 作为一个博客框架，他的主题属实有点少

当然可能还有其他的优缺点，请教万能的[百老师]，今天主要就是谈论一下如何安装hugo，基本配置以及远程发布到github

准备工作：

1.自行注册github账号，提前安装好git（自行百度，或者等待我下一篇git安装及基本操作，如果是windows以下操作最好在git bash中操作（git bash与linux操作相同），也可以自行下载chocolately，linux下直接命令行即可）

2.hugo下载（根据电脑系统选择下载方式）

  • windows：直接去官网下载 hugo中文网上面有更系统的教程（可对比学习），自认为本人写的更加通俗易懂

  • linux：命令行输入(brew install hugo）验证：hugo version

3.安装成功后创建一个站点

  • hugo new site myblog

4.查看[ls]当前目录下就会生成一个myblog文件夹目录结构如下

  • archetypes 网站主要页面

  • content 发布的博客内容（在其中的每个顶级文件夹都被看做一个内容部分）

  • data 数据信息

  • layouts 布局信息

  • public 发布的静态页面资源目录 需要输入命令hugo才会创建

  • resources 资源目录

  • static 静态的资源

  • themes 下载的主题

  • config.toml 采用toml格式编写的配置文件

5.下载主题

  • hugo中文网 这个网页啥都有（英文版是themes）中文版就是皮肤列表

  • 点进喜欢的页面之后，（找到Installation）cd 到themes目录下 复制 （举例）git clone https://github.com/syui/hugo-theme-air 到bash中就会进行下载

  • 下载后查看当前目录的子目录 可以把archetypes整体复制到你创建站点目录下覆盖掉原有的archetypes目录

  • 将theme.toml同样复制到你创建站点目录下覆盖掉原有的config.toml(你需要对theme.toml进行重命名)

  • 如果不想修改也可以做下面的变更（复制俩个连接一个config.toml，另一个是default.md）

6.创建你的第一篇博文

  • hugo new post/博文名.md 下面会弹出创建的目录，细心的小伙伴会发现，其实市创建在content的post目录下的markdown文件 md文件即为markdown(根据模块内容可以适当创建其他类似post的模块)

  • 在运行博客之前，我们要先确定一件事在config.toml中是否有baseUrl的属性，这个是博客发布的地址，默认和手动改为https://localhost：1313 是一样的（后面需要对其修改，修改为github地址），确定是否有theme属性，在其后指定为你的下载的主题名（即目录名）

  • 上述操作可以通过toml 进行操作，也可以通过json进行操作，还有一种yaml（小伙伴可以进行补充）

7.创建好博文后，需要进行发布，发布方式有很多种

  • 可以通过hugo初始化public

  • 发布博客下面会有提示信息端口号为1313 hugo server -t 主题名 –buildDrafts 指定主题名进行渲染 buildDrafts强行渲染

  • hugo server 这个需要确定baseUrl和theme就是第六点中谈到的，也就是说 配置好后就不需要强行进行渲染和 指定主题的操作

  • 发布后就会发现下面提示你去localhost：1313，登录这个网址你就会看到你的博客主页以及刚才编写的信息

8.完成以上步骤基本博客就搭建完毕了，当然我们还需要将博客部署到github中

  1. 选择github页面类型 即（gitPages）选择用户组即可（默认）用户/组织页面（https://.github.io/）

  2. 在该存储库下创建.github.ioGitHub存储库

  3. 将远程仓库克隆到本地

    git clone <YOUR-PROJECT-URL> && cd <YOUR-PROJECT>

  4. 将创建好的hugo项目粘贴到本地仓库（即克隆后出现的目录）

  5. hugo构建项目

  6. hugo server 启动项目 去网页看一眼满意之后， ctrl+c停止 将生成的public完全删除（rm -rf public）

  7. 创建一个git子模块

    git submodule add -b master git@github.com:<USERNAME>/<USERNAME>.github.io.git public

  8. 放入脚本deploy.sh（作用就是git添加提交推到远程库中）

#!/bin/sh

# If a command fails then the deploy stops
set -e

printf "\033[0;32mDeploying updates to GitHub...\033[0m\n"

# Build the project.
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

# Go To Public folder
cd public

# Add changes to git.
git add .

# Commit changes.
msg="rebuilding site $(date)"
if [ -n "$*" ]; then
	msg="$*"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master

9.创建好之后要为该脚本赋予权限chmod+x deploy.sh

10.直接运行./deploy.sh即可,这样基本一个小型博客就搭建完成

总结：

  1. 需要提前掌握git基本命令喔！！
